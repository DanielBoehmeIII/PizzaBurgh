# Abel Visual Gaps

## Latest Graphify Reference Finding

The default reference graph shows Orbital Navigation as the most connected visual abstraction across the 11 default references.

Implication:

- Abel needs a stronger reusable orbital/navigation language.
- Orbital systems should not only decorate pages.
- They should structure navigation, page identity, node selection, exhibit browsing, and quest/skill progression.
- The next visual redesign should prioritize MainPage, GraphPage, and ExhibitionPage using this orbital/central-hero composition language.

## Last reviewed

May 7, 2026

## Global Problems

- UI is too dim in many places.
- Labels and microtext are often unreadable.
- Several pages visually collapse when WebGL is weak or absent.
- Too many elements still feel like CSS panels over a wallpaper instead of part of one world.
- Need stronger foreground / midground / background depth.

## Page Gaps

### MainPage / Portal

- Needs reliable cube/portal monument.
- Current page can feel like text floating over atmosphere.
- Needs stronger right-side instrument panel.

### GraphPage / Core Graph

- Needs stronger central selected node.
- Needs luminous edge hierarchy.
- Should feel like an intelligence field, not scattered dots.

### ExhibitionPage / Memory Exhibition

- Fix title/sidebar clipping.
- Central crystal must be larger and more ceremonial.
- Memory nodes need to feel like curated exhibits.

### FocusPage

- Still feels like a form/session setup page.
- Needs augmented-mind centerpiece/dashboard structure.

### EggHatchPage

- Too empty.
- Egg/reward object must become dramatic and central.

### ArchivePage

- Render or strip markdown syntax.
- Maintain editorial chamber style.

### SettingsPage

- Needs subtle Abel-world structure.
- Should be quiet but not unfinished.
